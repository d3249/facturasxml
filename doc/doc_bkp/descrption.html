    Este sistema permitirá analizar un directorio en busca de Facturas y crear Reportes personalizados.

    Una Factura es un archivo XML con una estructura definida por el Sistema de Administración Tributaria (SAT) de México. Esta estructura puede variar de emisor a emisor (específicamente la ausencia del namespace cfdi: en algunos). En general se puede consultar en el archivo doc/arbol.

    Los Reportes serán arvhivos CSV separados por tabuladores.

    El sistema tendrá dos interfaces con el usuario:

    La primera será por línea de comandos. La segunda será un front-end gráfico para la primera.

    Las opciones serán las siguientes:

    logfile_name:  Nombre del archivo de salida. Valor por default LOGFILE.log
summary_name:  Nombre del reporte generado. Valor por default RESUMEN.csv
columns:       Lista de columnas que aparecerán en el reporte. Valor por default archivo|fecha|nombre_emisor|monto
total:         Bandera lógica que indica si se desea una línea final con el total. Valor por default VERDADERO

    Los argumentos serán los siguientes:

    directory:     Directorio a analizar. Valor por default . (current directory)
